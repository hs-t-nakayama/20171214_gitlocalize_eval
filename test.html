Oracle Service Cloud

The May 2017 Oracle Service Cloud release adds new capabilities to Web Customer Service, Contact Center, Knowledge Management, Policy Automation, and Platform.

Web Customer Service

A reputation for excellent customer service can differentiate the brand and drive revenue. Oracle Web Customer Service helps organizations earn that reputation by transforming online web service into a highly interactive customer experience.

Community Self-Service Enhancements

Each release of Community Self Service after its August 2015 MVP launch provides additional “forum functionality” to build out the feature set and make community participation, moderation, and administration easier and more effective.

Forum Functionality - Phase 5

This current release includes several community administrative and back-end enhancements, as well as functionality for web users. The main new feature is a community home page which is being added to the reference implementation. This page, which acts as a "front door" to the community, contains both new and existing widgets to give users a high level, at-a-glance, view of community and other relevant activity. Anchoring the home page is a new widget that lists the community forums (i.e., products or categories, depending on how the community has been configured) and the activity within them (with controls to let the admin choose to display these forums dynamically, or specify exactly which ones appear). This package also includes enhancements to discussion and account management pages, as well as the workflow for choosing profile pictures—such as letting members use their Facebook profile picture as their community avatar—which builds off the avatar features from the previous two releases.

  • Modify the community announcement text

Customer Portal - CAPTCHA

The Customer Portal team has updated the CAPTCHA widget, by implementing the newest Google CAPTCHA service "reCAPTCHA" which is superior to the old CAPTCHA widget. By using the “reCAPTCHA” widget we’ve also improved accessibility capabilities.

Cross Channel Contact Center

Oracle Service Cloud Cross Channel Customer Service powers organizations to provide quick, consistent, proactive, and reactive assisted service across multiple channels. Oracle Service Cloud Cross-Channel Contact Center helps organizations increase efficiency and reduce operational costs by enabling customers to resolve simple issues using self-service, while directing the more complex issues to contact center agents.

Agent Browser Client

Oracle Policy Automation

Oracle Policy Automation is now fully integrated in the Agent Browser Client. Policy Automation enables agents to execute policy interviews within an Agent Browser Client incident workspace.

Analytics Runtime Enhancements

The May 2017 release brings several enhancements to reporting within the Agent Browser UI. Inline Edit has been added which enables users to edit the record data directly in reports without the need to open reports. This is will be a significant time saver, as multiple record fields in a report can be edited directly without having to open each record report separately.

In addition to Inline Edit, Cross Tabs reports are now supported in the Agent Browser Client. These are pivot table-like displays that allow users to group data by row and column output.

Additional report export formats are available and dashboard reports can now be exported from Service Cloud.

Steps to Enable

To inline edit content in Agent Browser UI, your administrator must ensure that the report you use includes inline edit permissions. In addition, you must have profile permissions to edit records.

Key Resources

  • Inline edit report content

    Knowledge Management

    Knowledge management is essential for delivering exceptional customer experiences and increasing customer loyalty. In the contact center, empowered, efficient agents must have access to timely information when they need it. Customers expect to find answers to their questions quickly and easily, when and where they ask them. Oracle Service Cloud Knowledge Management provides a fully integrated knowledge management solution that delivers a seamless, consistent experience across all channels, from self-service to contact centers to communities and social sites.

    Oracle Service Cloud Knowledge Advanced

    With the May 2017 release, Oracle continues to add new capabilities to the Knowledge Advanced edition, including new API functionality, and new features that enhance the knowledge experience for administrators, authors, agents, and self-service users.

    Knowledge Advanced: Multi-Language Search Based on User Preferences

    This feature limits search results to the languages that are selected in users’ profiles. This enhancement ensures that agent desktop users can view only the answers that are assigned to their authorized locales.

    Steps to Enable

    Application administrators can enable this feature by setting a parameter in the site configuration settings.

    Key Resources

    • See Configuring Searches to Filter by User Locale in the Configuring Agent Desktop section of the Knowledge Advanced Implementation Guide.

      Knowledge Advanced: REST API Improvements

      The May 2017 release adds new API functionality that facilitates importing images in bulk. Knowledge managers can upload and delete images in batches using the new API as an alternative to using the authoring tool. The API enables customers to mass update content with inline images, and also assists with migrating documents from other environments, such as an Enterprise Knowledge on premise implementation or a customer's legacy knowledge base.

      Steps to Enable

      Using the Knowledge API requires custom programming. Consult with your implementation specialists.

      Tips and Considerations

      Use the new Media endpoint to programmatically import images in bulk.

      Key Resources

      • Knowledge REST API endpoints are documented in REST API for Oracle Service Cloud Knowledge Advanced.

        Knowledge Advanced: Browsing by Product and Category

        This customer-requested feature improves user experience and productivity by enabling Customer Portal users to view answers of all content types in a single browse list. Self-service users can take a product- and category-driven approach to locating the information they need regardless of the type of answer that contains the solution.

        Steps to Enable

        Customer Portal developers can add the All option to enable users to view all content types in a single answer list.

        Tips and Considerations

        Product and category filters remain enabled, providing customers with the ability to view all content types filtered by the desired products and categories.

        Key Resources

        • See Enabling Users to Display Answers for All Content Types in the Enabling Knowledge Advanced on Customer Portal section of the Knowledge Advanced Implementation Guide.

          Knowledge Advanced: Authoring and Image Management Enhancements

          With the May release, the Knowledge Advanced document attachment management facility has been expanded to support any customer-specified file extension, such as PDF or .docx files. This feature simplifies authoring and enhances the agent experience by enabling authors to easily upload and link to non-image assets. In addition, the site-wide Link Checker now validates Knowledge Advanced answer links, enabling knowledge managers to easily identify and fix any broken links within the knowledge base.

          Key Resources

          • See the Service Cloud User Guide for information on link check reporting.

            Policy Automation

            Policy Automation enables the automation of an organization’s business rules, processes, policies, or regulations. With Policy Automation, organizations can ensure polices and regulations are being met, leverage intelligent decisioning, capture the right customer information in interviews, assist the resolution of complex service requests, and help determine product or service eligibility.

            Generate Documents Using PDF Templates

            Form templates can now be maintained as PDF files. You can use any PDF editing tool of choice to modify existing form templates, or to design new ones, and fill the form with data and decisions from an OPA interview. Forms generated from PDF templates are treated just like forms generated from RTF templates. That is, you can provide download links for customers to save and print forms, as well as attach forms to records in Service Cloud or submit forms to other applications via a web service connector.

            Steps to Enable

            1. In Policy Modeling, on the Projects tab, select the Forms subtab.
            2. Add a new PDF form, or import an existing PDF form.
            3. Open the PDF form template in your preferred PDF editor.
            4. In form edit mode, prepare the form template using the OPA PDF Form Assistant to add attribute text, attribute values and explanations.
            5. Add the form to a screen in an OPA interview.

            Tips and Considerations

            • A PDF form template is now included in the Healthy Eating example project installed with policy modeling.

                Interview Extensions

                JavaScript-based interview extensions are now available making it possible to develop and maintain upgradable extensions for all OPA projects. Extensions allow you to restyle existing interview components and provide custom rendering of interview components. You can use any control you can imagine, such as maps, menus or calendars, and apply you own corporate styles to any interview component.

                Steps to Enable

                1. On the Interview tab in Policy Modeling, click the Styles button.
                2. In the Styles dialog box, click the Custom Files button. Windows Explorer will open to the /interview-theme/resources/ folder that has been created in the project directory.
                3. Create the interview extension .js file in the /interview-theme/resources/folder.
                4. In the interview extension .js file, use the OraclePolicyAutomation.AddExtension() method to specify the desired features and handlers. In an interview, the interview extension JavaScript file will be automatically loaded for every page in the OPA interview.

                Tips and Considerations

                1. JavaScript extensions can reference images and other resource files, as well as custom stylesheets, added to the policy model as custom files.
                2. Extensions work with both web and mobile OPA interviews.
                3. Interview extensions are demonstrated in the DMV Document Guide, Loan Advisor and Path Finder example policy models installed with Policy Modeling.

                Embeddable Interviews

                New, in this release, is the capability to embed interviews on any HTML page without the need for iframes, by using the provided script and stylesheets. By referencing styles from the parent page's stylesheet, you can comply with any corporate branding requirements.

                You can also control which web sites any deployed interview can be embedded within. This ensures maximum compatibility and security of OPA interviews that are integrated into other application experiences.

                Steps to Enable

                To embed an OPA interview on a web page:

                1. Add the metadata to link in the required CSS and JavaScript files.
                2. Add a <div> where you would like the OPA interview to appear.
                3. Add the JavaScript to start (or resume) the OPA interview. The method to start an interview is: OraclePolicyAutomationEmbedded.StartInterview(el, webDeterminationsUrl, deploymentName).

                To allow permitted sites to embed deployed OPA interviews:

                1. Log in to the Policy Automation Hub web interface.
                2. On the Deployments tab, click the Actions drop-down menu and select Authorize Embedded Interviews.
                3. In the Authorize Hosts window, enter the addresses for any authorized hosts.

                Tips and Considerations

                • It is possible to show multiple OPA interviews on a single web page. This can be useful for helper interviews within a longer advice session. Each interview can take on separate styling but any interview extensions are shared.

                    Batch Assess REST API

                    The new Batch Assess REST API efficiently processes multiple interviews in bulk. This allows interviews to apply rules and policies consistently with both interactive and batch processes. With this new feature, organizations can improve service quality with regular bulk reassessment of data such as case or incident status or other decisions critical to the business.

                    Steps to Enable

                    1. Prepare case data in JSON format.
                    2. Provide an ID for each one.
                    3. Specify which outcomes are needed.
                    4. POST the request to assessor URI.
                    5. Outcomes are returned for each case.

                    Tips and Considerations

                    • Each REST API call can pass up to 10MB of assessment data (this can be increased if needed)
                    • All resources in the REST API are self-describing, with sub-resource URIs provided at every level.
                    • Errors and warnings are detailed inline for affected cases and have no affect on successful cases in the same batch.
                    • OAUTH2 is used for authentication.
                    • A new OPA Assessment API SKU is planned (for the June 2017 price list)
